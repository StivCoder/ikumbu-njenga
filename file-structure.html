The Files Explained

HTML
index.html
stiv-tech-solutions/index.html
The Home page. This is the first file a browser loads. It defines the page structure and content using HTML tags.
  • <nav> — navigation bar
  • <section class="hero"> — hero banner with photo
  • <div class="tech-ticker"> — scrolling skills ticker
  • <section class="skills-section"> — tech stack cards
  • <section id="projects"> — live project previews
  • <div class="stats-bar"> — stat numbers
  • <footer> — site footer
  • Links to css/style.css & js/main.js
HTML
about.html
stiv-tech-solutions/about.html
The About page. Contains your story, photos, tech stack, timeline, and values — same HTML structure as index but different content.
  • .about-dual-photos — both your photos displayed
  • .about-meta — name, role, location info
  • .tech-badges — hoverable skill badges
  • .timeline — career journey items
  • .values-grid — four value cards
  • .about-cta — call-to-action block
  • Links to same css/style.css & js/main.js
HTML
contact.html
stiv-tech-solutions/contact.html
The Contact page. Contains your contact details and a working message form. Calls handleSubmit() from main.js on submit.
  • .contact-info — email, phone, location
  • .availability — pulsing green status
  • #contactForm — the message form
  • #formSuccess — success state (hidden until submit)
  • onsubmit="handleSubmit(event)" — calls JS function
CSS
style.css
stiv-tech-solutions/css/style.css
The master stylesheet. Controls ALL visual design — colors, fonts, layout, animations, hover effects, and responsive rules for every HTML element across all pages.
  • :root { } — design tokens (colors, sizes)
  • nav { } — navigation styles
  • .hero { } — hero section layout (CSS Grid)
  • @keyframes — ticker, pulse, fillBar animations
  • @media (max-width:900px) — mobile responsive rules
  • .reveal — scroll animation base class
JavaScript
main.js
stiv-tech-solutions/js/main.js
The master script. Adds all interactivity — hamburger menu, scroll effects, reveal animations, cursor dot, smooth scrolling, and form submission.
  • DOMContentLoaded — waits for page to load
  • hamburger.addEventListener — toggles mobile menu
  • IntersectionObserver — triggers scroll animations
  • navbar.classList.toggle('scrolled') — shrinks nav
  • mousemove — moves cursor dot
  • handleSubmit() — processes contact form
Images
images/
stiv-tech-solutions/images/
The images folder. Stores all photo assets used by the HTML pages. Referenced using relative paths like src="images/photo1.jpg".
  • photo1.jpg — portrait photo (hero + about)
  • photo2.jpg — waterfall adventure photo (about)
  • Add more images here as your portfolio grows
  • Use descriptive names: project-school.jpg

HTML — Structure

HTML is the skeleton of every page. It defines what content exists and where it goes, using tags as building blocks.

index.html — head section <!DOCTYPE html> ← tells browser: "this is HTML5" <html lang="en"> ← root element, lang for accessibility <head> ← invisible metadata <meta charset="UTF-8"/> ← character encoding <meta name="viewport" content="width=device-width"/> ← responsive <title>Stiv Tech Solutions</title> ← browser tab text <link rel="stylesheet" href="css/style.css"> ← loads CSS </head> <body> ← everything visible goes here <nav>...</nav> <section>...</section> <footer>...</footer> <script src="js/main.js"></script> ← loads JS last </body> </html>
Semantic HTML Relative paths Meta tags External resources DOM tree Block vs Inline

CSS — Styling

CSS is the skin of the website. It controls colors, fonts, layout, spacing, animations, and responsive behaviour.

css/style.css — key concepts /* 1. CSS Custom Properties (variables) */ :root { --accent: #00ff88; ← define once, use everywhere --bg: #0a0a0f; --text: #e8e8f0; } /* 2. Selectors target HTML elements */ nav { position: fixed; } ← element selector .hero { display: grid; } ← class selector #projects { padding: 6rem 4rem; } ← ID selector /* 3. Animations with @keyframes */ @keyframes ticker { from { transform: translateX(0); } to { transform: translateX(-50%); } } /* 4. Responsive — mobile breakpoints */ @media (max-width: 900px) { .hero { grid-template-columns: 1fr; } ← stack on mobile }
CSS Variables Flexbox CSS Grid @keyframes Media Queries Pseudo-elements Transitions clip-path

JavaScript — Behaviour

JavaScript is the brain. It makes the website interactive — responding to clicks, scroll events, and user input in real time.

js/main.js — key concepts /* 1. Wait for page to load before running */ document.addEventListener('DOMContentLoaded', () => { /* 2. Select elements from the DOM (HTML tree) */ const hamburger = document.getElementById('hamburger'); const navLinks = document.getElementById('navLinks'); /* 3. Listen for events (click, scroll, mousemove) */ hamburger.addEventListener('click', () => { navLinks.classList.toggle('open'); ← add/remove CSS class }); /* 4. IntersectionObserver — detect when element enters viewport */ const observer = new IntersectionObserver((entries) => { entries.forEach(entry => { if (entry.isIntersecting) { entry.target.classList.add('visible'); ← trigger CSS animation } }); }); }); /* 5. Form handler — called from HTML onsubmit attribute */ function handleSubmit(e) { e.preventDefault(); ← stop default form reload window.location.href = mailtoUrl; ← open email client }
DOM Manipulation Event Listeners classList.toggle IntersectionObserver Arrow Functions Template Literals encodeURIComponent DOMContentLoaded
// SUMMARY

HTML = content & structure  |  CSS = visual design & layout  |  JavaScript = interactivity & behaviour

All three files are kept separate (separation of concerns) so each does its own job cleanly. Every HTML page links to the same css/style.css and js/main.js — one change in the CSS instantly updates all three pages.
